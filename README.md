# בונה פרומפט תכנון מערכת — by Jack Vidal

> סקיל ל‑Claude שעוזר למי שאין לו רקע טכני לבנות **פרומפט תכנון מערכת מקצועי** — בעברית מלאה.

---

## מה זה? (בשפה פשוטה)

נניח שיש לך רעיון למערכת או אפליקציה (CRM, חנות, מערכת לקביעת תורים, כל דבר) —
אבל אתה לא מתכנת, ואתה לא יודע איך "לבקש מ‑Claude" לבנות אותה בצורה נכונה.

הסקיל הזה פותר בדיוק את זה. הוא **מראיין אותך**: שואל אותך שאלות פשוטות על העסק
והרעיון שלך (שאלה אחת בכל פעם), מסביר לך כל מושג טכני במשפט אחד תוך כדי, ובסוף
**מרכיב עבורך פרומפט תכנון מלא ומקצועי** — מהסוג שמבקש מ‑Claude לתכנן את המערכת
לפני שכותבים שורת קוד אחת.

את הפרומפט המוכן אתה מעתיק לשיחה חדשה עם Claude, ומקבל תכנון מלא של המערכת:
ארכיטקטורה, בסיס נתונים, מבנה קבצים, סדר פיתוח ועוד.

**זה "שלב 0" של בניית מערכת** — לפני שמתכננים, ובוודאי לפני שבונים.

> ⚠️ הסקיל הזה **לא כותב קוד ולא בונה את המערכת**. הוא מפיק רק את פרומפט התכנון.

---

## מה אתה מקבל בסוף

פרומפט עברי מסודר שכולל:

- תיאור המערכת, המשתמשים, והדברים שהיא מנהלת
- הפעולות, הסטטוסים, פיצ'רי ה‑AI והאינטגרציות (אם יש)
- דרישות ממשק, עברית/RTL, ואבטחה
- בקשה ל‑Claude לתכנן: ארכיטקטורה, Stack מומלץ, מבנה תיקיות, מודלי נתונים, קשרים,
  API, משתני סביבה, סדר פיתוח, אתגרים צפויים, ומה צריך בגרסה ראשונה

## איך מתקינים

הסקיל הזה הוא **Claude Code Skill** — תיקייה עם קובץ `SKILL.md`. מתקינים אותו
פעם אחת, והוא נטען אוטומטית בכל פעם שתפתח את Claude.

### שלב 1 — לאן מתקינים (תיקיית הסקילים האישית)

| מערכת הפעלה | נתיב התיקייה |
|--------------|---------------|
| **Windows**  | `C:\Users\<שם המשתמש שלך>\.claude\skills\` |
| **macOS**    | `~/.claude/skills/` |
| **Linux**    | `~/.claude/skills/` |

> אם התיקייה `.claude\skills` עדיין לא קיימת — פשוט תיצור אותה.

### שלב 2 — להוריד את הסקיל לשם

יש שתי דרכים. בחר אחת:

**דרך א' — עם Git (הכי פשוט, אם מותקן אצלך git):**

```bash
# macOS / Linux
git clone https://github.com/jackvidal/system-planning-prompt-builder-by-jack-vidal \
  ~/.claude/skills/system-planning-prompt-builder-by-jack-vidal
```

```powershell
# Windows (PowerShell)
git clone https://github.com/jackvidal/system-planning-prompt-builder-by-jack-vidal `
  "$env:USERPROFILE\.claude\skills\system-planning-prompt-builder-by-jack-vidal"
```

**דרך ב' — בלי Git (הורדת ZIP):**

1. היכנס לעמוד הריפו בגיטהאב.
2. לחץ על הכפתור הירוק **Code** ← **Download ZIP**.
3. חלץ (Extract) את הקובץ.
4. העתק את התיקייה `system-planning-prompt-builder-by-jack-vidal` אל תוך תיקיית
   הסקילים מהטבלה למעלה.

> חשוב: בסוף התהליך הנתיב צריך להיראות כך —
> `…\.claude\skills\system-planning-prompt-builder-by-jack-vidal\SKILL.md`
> (כלומר `SKILL.md` יושב **ישירות** בתוך תיקיית הסקיל, לא בתוך תת‑תיקייה נוספת).

### שלב 3 — להפעיל

זה עובד לכל מי שמשתמש ב‑**Claude Code**, בכל צורה שבה הוא רץ:

- **בתוך ה‑IDE** (VS Code / Cursor / JetBrains, דרך תוסף Claude Code)
- **בטרמינל** (Claude Code CLI)

שתי הצורות קוראות מאותה תיקייה (`~/.claude/skills/`), אז ההתקנה זהה.

1. פתח **שיחה חדשה** ב‑Claude Code (או הפעל מחדש את ה‑IDE) כדי שהסקיל ייטען.
2. פשוט תאר מה אתה רוצה, למשל:
   > "יש לי רעיון לאפליקציה ואני לא יודע איך לבקש לתכנן אותה"
3. Claude יזהה את הסקיל ויתחיל לראיין אותך. אם אתה רוצה לוודא — אפשר גם לכתוב
   במפורש: *"תשתמש בסקיל system-planning-prompt-builder".*

## שימוש ב‑Claude Desktop

הסקיל בנוי בפורמט של **Claude Code** (קובץ `SKILL.md` בתיקיית `~/.claude/skills/`).
אם אתה עובד עם **Claude Code** (CLI או תוסף ל‑IDE) — ההתקנה למעלה היא כל מה שצריך,
וזו הדרך המומלצת לקורס.

אפליקציית **Claude Desktop** מנהלת "Skills" דרך ההגדרות שלה (Settings → Capabilities),
ולא תמיד דרך תיקיית `~/.claude/skills/`. אם אתה רוצה להריץ את הסקיל ספציפית בתוך
Claude Desktop, היכנס ל‑**Settings → Capabilities/Skills** באפליקציה ופעל לפי
האפשרות שם להוספת Skill (ייתכן שתצטרך להעלות את תיקיית הסקיל). הדרך הכי פשוטה
והבטוחה למתחילים נשארת **Claude Code ב‑IDE** לפי ההוראות למעלה.
